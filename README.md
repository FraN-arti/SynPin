## Что такое SynPin

SynPin — это рабочая среда, в которой **несколько ИИ-агентов совместно решают задачи под твоим руководством**. Главный агент координирует отделы специализации, ставит задачи, делегирует и следит за прогрессом. У каждого агента — долгосрочная память, роль и место в иерархии.

Система задумана для работы **между сессиями**: агенты сами ставят себе напоминания, эскалируют заблокированное, возвращаются к незаконченному. Это не чат-фреймворк и не набор плагинов. Это организация, которая живёт ровно столько, сколько ты ей позволяешь.

SynPin полезен тогда, когда задачи выходят за рамки одного разговора: проекты на недели вперёд, отчёты которые лучше делать ночью, идеи которые нужно довести, не сегодня так завтра. Там, где ты не хочешь каждый раз заново объяснять контекст.

<br/>

## Принципы

<span style="color:#f97316">**Агенты — это граждане, а не инструменты.**</span> У каждого есть роль, отдел, долгосрочная память. Они могут ставить себе напоминания, брать работу у соседей, эскалировать вверх. Это не «один агент делает всё» и не «набор ассистентов без памяти между собой».

<span style="color:#f97316">**Память — это собственное знание.**</span> У каждого агента три области: `USER` (кто ты), `MEMORY` (что он узнал), `FACTS` (что решили). Агент сам решает, куда положить факт, и сам его потом находит. Не search по векторной БД — это структурированное знание с историей.

<span style="color:#f97316">**Proactivity, а не реактивность.**</span> Агенты могут сами создавать cron-задачи, sweep'ить память, реагировать на события в каналах. SynPin запускает фоновый daemon manager, как в Linux — расписание, retention, автоматические действия. Это работает, пока компьютер не выключен.

<span style="color:#f97316">**Тёмная палитра и стеклянные панели.**</span> Интерфейс спроектирован для долгих сессий ночью — минимум контраста, минимум деталей. Тёплый фон, оранжевый акцент, только то что нужно прямо сейчас.

<br/>

## Возможности

- **Главный агент с координацией.** Делегирует задачи отделам, собирает сводки, общается с пользователем от имени системы.
- **Отделы специализации.** Департаменты с собственными агентами и контрактами. Например: «Общение», «Идеи», «Код», «Канбан».
- **Соединения между отделами.** Как `otdel:<id>` → `agent:primary` в коде; можно настраивать потоки задач между ними.
- **Долгосрочная память.** По каждому агенту — `USER.md`, `MEMORY.md`, `FACTS`. С retention и архивацией.
- **Cron для агентов.** Любой агент может поставить задачу с `target`, `action_target`, `delivery`, расписанием.
- **Канбан.** Доски задач с этапами, исполнителями, дедлайнами.
- **Проекты, сессии, факты, триггеры, плагины, скиллы.** Всё подключаемое.

## Установка

Одна команда на каждой платформе. Скрипт сам клонирует репо в `~/synpin/`, создаёт `.venv`, ставит Python-зависимости, билдит фронтенд. Папка одна.

**macOS / Linux:**

```bash
curl -fsSL https://raw.githubusercontent.com/FraN-arti/SynPin/main/install | bash
```

**Windows (PowerShell):**

```powershell
irm https://raw.githubusercontent.com/FraN-arti/SynPin/main/bootstrap.ps1 | iex
```

После установки открой новый терминал и запусти:

```
synpin start
```

UI на `http://localhost:2088`. Wizard проведёт через выбор провайдера и создание первого агента.

## Разработка

Если хочешь участвовать в проекте или форкнуть:

```bash
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin
./install.sh           # или install.ps1 на Windows
synpin dev              # backend + frontend с hot-reload
```

`dev` поднимает API на `:2088` и Vite на `:2099`. Изменения в `core/synpin/` подхватываются через uvicorn reload, изменения в `web/src/` — через Vite HMR.

Контрибьюторам: [CONTRIBUTING.md](CONTRIBUTING.md) описывает архитектуру, конвенции, AGENTS.md правила.
