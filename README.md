## Зачем это нужно

Существующие ИИ-фреймворки дают вам инструмент. Вы нажимаете кнопку — агент отвечает. Забыл контекст — начинай сначала. Хочешь чтобы два агента обсудили архитектуру — пиши промт вручную.

**SynPin — это другое.**

Здесь агенты живут в организации. У каждого есть имя, роль, департамент. Они знают где работают, помнят прошлые разговоры, используют инструменты и могут вызвать друг друга. Как настоящая команда.

## Что уже работает

### 💬 Живой чат с инструментами

Агент отвечает в реальном времени (SSE-стриминг). Но не просто текстом — он может **выполнять действия**:

```
Вы: "Создай файл config.py с настройками базы данных"

Агент:  🔧 file_write → config.py создан
        Вот что я написал...
```

**6 инструментов работают прямо сейчас:**
- 📂 Чтение и запись файлов
- 🔍 Поиск по кодовой базе
- 💻 Выполнение shell-команд и Python-скриптов
- 🌐 Поиск в интернете

И ещё 9 в разработке (браузер, память, делегирование между агентами).

### 🤖 Агенты с личностью

Каждый агент — не безликий бот, а сотрудник с характером:

| Параметр | Что даёт |
|---|---|
| **Имя и роль** | "Backend Lead" — советчик, "Frontend Developer" — кодер |
| **Департамент** | К какому отделу относится |
| **Тон и стиль** | Строгий / дружелюбный / формальный |
| **Системный промт** | Уникальные инструкции для каждого агента |
| **Температура** | Насколько креативно или точно отвечает |

### 🔌 Мульти-провайдеры

Один агент может работать через один провайдер, другой — через другой.

```yaml
# Пример конфигурации
providers:
  - name: 9router
    type: openai
    endpoint: http://localhost:20128/v1
    model: general-agent
```

Поддерживаются: **OpenAI-compatible** (через OpenRouter, 9router, Ollama...) и **Anthropic** (Claude).

### 🔗 Внешние агенты

Подключайте готовых агентов извне. Например, **Hermes Agent** — ИИ-ассистент с собственной памятью и навыками. Он автоматически получает контекст SynPin и работает как полноправный сотрудник.

### 🏗 Иерархия организации

```
SynPin
├── Роли:         Управляющий → Советчик → Сотрудник
├── Департаменты: Backend → Frontend → Analytics
└── Агенты:       Backend Lead, Frontend Developer, QA Engineer...
```

Каждый агент принадлежит роли и департаменту. Это не просто метки — это структура для будущей навигации и управления.

### 🧠 Память пользователя (USER.md)

Каждый агент автоматически видит информацию о пользователе в системном промпте. USER.md — глобальный для всех агентов:

```
USER PROFILE (who the user is):
👤 Имя: Артур — разработчик, Москва
🐙 GitHub: FraN-arti
📧 Email: artur.kalish.90@gmail.com
```

Агенты пишут в USER.md через memory_write, вы читаете в UI. Автоматический онбординг: если USER.md пуст, агент сначала представляется и спрашивает имя/роль.

### 🔧 Реестр инструментов

Инструменты группируются по категориям:

| Категория | Инструменты |
|---|---|
| **Файлы** | Чтение, запись, поиск |
| **Код** | Терминал, запуск Python |
| **Веб** | Поиск, браузер* |
| **Память** | Глобальный USER.md, MEMORY.md*, чтение*, запись* |
| **Коммуникация** | Отправка сообщений*, вызов агентов* |
| **Скиллы** | Применение* |

\* — в разработке

### ⚙️ Настройки через UI

Всё управление — в одном месте:
- Создание и удаление агентов
- Назначение ролей и департаментов
- Подключение провайдеров
- Управление инструментами (вкл/выкл для каждого агента)
- Настройка личности агента

## Быстрый старт

### Установка

```powershell
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin
pip install -e ./core
```

### Запуск

```powershell
synpin start
```

Откройте `http://localhost:2088` — ваша организация готова.

### CLI

| Команда | Описание |
|---|---|
| `synpin start` | Запуск сервера |
| `synpin stop` | Остановка сервера |
| `synpin status` | Проверка статуса |
| `synpin setup` | Мастер начальной настройки |
