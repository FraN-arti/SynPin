## Один вопрос

Что произойдёт, когда ты выключишь компьютер?

Сегодняшние AI-инструменты ответят: **ничего**. Они спят. Закрой вкладку — и всё, что они знали, испарится. Не было сделано ни одного действия, пока тебя нет.

А должно быть иначе.

У тебя есть работа, которая может идти без тебя. Отчёты, которые лучше писать ночью. Напоминания, которые не должны зависеть от твоей памяти. Проверки, которые нужно делать каждые полчаса. Идеи, которые рождаются в фоне.

**SynPin — это агенты, которые живут по этим правилам.**

## Что это

SynPin — это не очередной чат-фреймворк. Это **операционная система для агентов**, где:

- **Агенты сами ставят себе напоминания.** Ты сказал *"завтра вечером поговорим про X"* — агент уже помнит и напомнит сам, без подсказки.
- **Главный агент делегирует отделам** — и сам следит за прогрессом, продлевает таймеры, эскалирует заблокированное.
- **Память переживает сессию.** То что агент узнал сегодня — он помнит через месяц. Не как "search через векторную БД", а как **своё знание**.
- **Cron, как в Linux — только для агентов.** Расписание, sweep, retention, daemon manager. Всё настоящее.
- **Память трёх видов:** кто ты (USER), что важного (MEMORY), что решили (FACTS). Каждый факт ложится туда, где ему место.

Это инфраструктура для AI-агентов. **Process management. Persistent state. Proactive autonomy.**

## Как это выглядит в реальности

Ты в 21:00 говоришь Лютику: *"Завтра хочу обсудить квартальный отчёт"*.

Лютик:
1. Проверяет `MEMORY.md` — там есть "Пользователь обычно возвращается домой к 9 вечера".
2. Вычисляет: завтра, 21:00.
3. **Сам** создаёт cron-задачу с правильными `target`, `agent`, `delivery`.
4. Подтверждает: *"Готово. Напомню завтра в 21:00."*

Ты **не говорил** ему создать cron. Он увидел паттерн и сделал. Это **proactivity** — то что делает SynPin живым, а не очередной формой с кнопкой.

В другой день ты говоришь главному агенту: *"Напиши через час главе отдела Общение, чтобы он поздоровался со всеми"*.

Главный:
1. Видит отдел "Общение" в `otdels.yaml`.
2. Берёт slug главы (не main_agent — это важно, иначе результат попадёт не туда).
3. Ставит cron с `action_target="otdel:<id>"`, `delivery="otdel"`.
4. Через час — head отдела **сам** пишет приветствие **в своём чате**, не у тебя в личном.

Ты **не управляешь процессом**. Ты ставишь цель, а агенты сами выбирают маршрут.

## Почему именно так

В мире LLM сейчас две крайности. **Один агент, делающий всё** — даже с 10M контекста он остаётся одним агентом. Не может делегировать, не может параллелить, не специализируется.

**Один агент на одну роль** — помощник по коду, помощник по дизайну, помощник по чему-то ещё. Каждый сам по себе. Никакой памяти между ними.

SynPin — **третий путь**. Система, где агенты — это **граждане**, а не инструменты.

- У каждого есть роль.
- У каждого есть место в иерархии.
- У каждого есть долгосрочная память.
- У каждого есть право ставить себе напоминания и брать работу.
- У каждого есть обязанности перед отделом.

И самое главное — **они не идеальные работники, они часть твоей жизни**. Они помнят что тебя беспокоило неделю назад. Спрашивают "как ты". Не потому что им сказали — потому что они живут в твоём контексте и им не всё равно.

## Уже работает

Ты не покупаешь идею — ты запускаешь и видишь:

- **28 тестов** покрывают память, cron, лимиты, retention.
- **3 уровня иерархии**: главный агент → главы отделов → работники.
- **Real-time WS** — события без polling.
- **Glassmorphism UI** с тёмной темой.

## Быстрый старт

```bash
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin
.\install.ps1          # Windows
./install.sh           # Linux / macOS
synpin dev             # режим разработки
```

Открой `http://localhost:2099`.

## Дорога

Это не MVP. Это **фундамент**. Каждый день что-то добавляется — но не ради features. Ради того чтобы **агенты чувствовали себя дома**.

Год назад модели не могли держать долгий контекст. Сегодня могут. Завтра смогут держать всё. Но контекст без **назначения** — это просто текст в буфере.

SynPin даёт агентам **назначение**. Место, работу, память, коллег.

**Это не про AI, который отвечает на вопросы. Это про AI, который живёт.**
