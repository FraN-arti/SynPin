## Зачем это нужно

Существующие ИИ-фреймворки дают вам инструмент: нажал кнопку — агент ответил. Забыл контекст — начинай сначала. Хочешь, чтобы два агента обсудили архитектуру — пиши промт вручную.

**SynPin — другое.** Здесь агенты живут в организации. У каждого есть имя, роль, департамент. Они знают, где работают, помнят прошлые разговоры, используют инструменты и могут вызвать друг друга. Как настоящая команда.

## Возможности

### 💬 Чат с инструментами

Агенты отвечают в реальном времени (SSE-стриминг) и выполняют действия: читают и пишут файлы, запускают команды, ищут в интернете, работают с памятью. Каждое действие видно в интерфейсе.

**13 инструментов работают:**
- 📂 Чтение, запись, поиск файлов
- 💻 Терминал, запуск Python-скриптов
- 🌐 Поиск в интернете
- 🧠 Чтение и запись памяти (USER.md, MEMORY.md)
- 🔄 Делегирование Head Protocol (5 инструментов)

### 🏢 Организационная структура

Департаменты → Отделы → Агенты. Двухуровневая иерархия для реальной рабочей структуры.

```
SynPin
├── Роли:         Управляющий → Работник отдела
├── Департаменты: Разработка, Волокита, Вахтан, Сузумебачи
├── Отделы:       Грахатули, Обсуждение аниме
└── Агенты:       QA Инженер, Архитектор, Кодер, Тестер...
```

### 🧑‍💼 Агенты с личностью

Каждый агент — сотрудник с характером: имя, роль, тон общения, температура, системный промт. Не безликий бот, а коллега со специализацией.

### 🔄 Делегирование (Head Protocol)

Управляющий ставит задачи работникам, ждёт ответов, оценивает результаты и принимает решения. Настоящая иерархическая коммуникация через 5 инструментов:

- `head_delegate` — постановка задач
- `head_await` — ожидание ответов
- `head_evaluate` — оценка результатов
- `head_retry` — повтор при ошибке
- `head_decide` — стратегическое решение

### 🧠 Память агентов

Каждый агент хранит MEMORY.md — компактную память о прошлых разговорах, решениях и ошибках. Контекст компактируется, но важное остаётся.

### 🔌 Мульти-провайдеры

Разные агенты работают через разные LLM. OpenAI-compatible (OpenRouter, Ollama, локальные модели) и Anthropic (Claude).

### 🔗 Внешние агенты

Подключайте готовых агентов извне. Hermes Agent автоматически получает контекст SynPin и работает как полноправный сотрудник.

### 🎨 Темы

Система тем с импортом из [tweakcn.com](https://tweakcn.com) — настраивайте внешний вид под себя.

### 📊 Статистика

Просмотр использования: общая статистика, загрузка агентов, использование инструментов, история сессий.

### ⚙️ Настройки через UI

Всё управление в одном месте: агенты, роли, департаменты, отделы, провайдеры, инструменты, темы.

## Быстрый старт

### Установка

```powershell
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin
pip install -e ./core
```

### Запуск

```powershell
synpin start
```

Откройте `http://localhost:2088` — ваша организация готова.

### CLI

| Команда | Описание |
|---|---|
| `synpin start` | Запуск сервера |
| `synpin stop` | Остановка сервера |
| `synpin status` | Проверка статуса |
| `synpin setup` | Мастер начальной настройки |
