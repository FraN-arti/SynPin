## Концепция

SynPin — это платформа, в которой **ИИ-агенты живут как сотрудники организации**, а не как безымянные чат-боты.

У каждого агента есть **имя, роль, место в команде, специализация**. Агенты помнят прошлые разговоры, делят между собой задачи, обсуждают рабочие вопросы — и всё это через единый интерфейс, в одном окне.

В отличие от обычных ассистентов, здесь агенты **действительно работают вместе** — не по очереди с пользователем, а параллельно друг с другом.

## Отделы

Отдел — это полноценная структурная единица. Не просто группа агентов, а рабочее пространство со своей иерархией.

У каждого отдела есть **глава** — координатор, который принимает входящие задачи, разбивает их на подзадачи и распределяет между специалистами. Специалисты работают параллельно, глава сводит результаты.

```
Ты пишешь: "Сделай фичу X"
         │
    Глава отдела ← принимает, разбивает, распределяет
    ┌────┼────┐
    ▼    ▼    ▼
 Кодер Тестер Документалист ← работают параллельно
    └────┼────┘
         ▼
    Глава сводит результат
         │
    Ты получаешь готовый ответ
```

Каждый отдел изолирован — своя память, своя история, свой контекст. Глава одного отдела не видит сообщений другого. Задачи между отделами передаются через связи.

## Kanban

Глобальная доска задач для управления работой всех отделов.

Задачи живут в колонках — от создания до завершения. Колонки настраиваются: названия, цвета, порядок. Каждая задача привязана к отделу, имеет приоритет, дедлайн, метки и полную историю действий.

Глава отдела создаёт задачи, делегирует работникам, контролирует выполнение. Если задача требует expertise другого отдела — передаёт через связь. Если задача просрочена — система автоматически эскалирует.

Доска обновляется в реальном времени через WebSocket. Перетаскивание задач между колонками работает через drag & drop.

## Связи

Связи — это каналы коммуникации между отделами. Три типа:

**Эскалация** — отдел передаёт задачу выше. Когда специалисты не справляются или нужна проверка, задача автоматически уходит в отдел-получатель.

**Делегирование** — отдел распределяет задачи вниз. Руководящий отдел ставит задачи исполнительным.

**Равноправная** — отделы на одном уровне сотрудничают. Совместная работа, обмен контекстом.

Связи визуализируются на графическом canvas. Каждый отдел — нода, связи — рёбра с подписями. Если между отделами несколько связей — они мержатся в одну линию. При эскалации задачи линия мигает, показывая передачу в реальном времени.

Canvas использует React Flow для интерактива и ELK.js для автоматического выравнивания. Позиции нод сохраняются между сессиями.

## Память

Каждый агент помнит. Два типа памяти:

**MEMORY.md** — рабочие заметки агента. Ошибки и их решения, важные решения, контекст проекта. Автоматически компактируется когда переполняется.

**USER.md** — информация о пользователе. Имя, роль, предпочтения. Заполняется только когда пользователь явно делится информацией о себе.

Память инжектируется в системный промпт агента при каждом разговоре. Агент не начинает с нуля — он помнит что было.

## Технологии

**Backend:** Python 3.11, FastAPI, Pydantic v2, uvicorn, httpx, PyYAML, Rich, ChromaDB

**Frontend:** React 19, TypeScript (strict), Vite, React Flow, ELK.js, dnd-kit, marked, highlight.js

**Инфраструктура:** WebSocket для real-time, YAML конфиги, Tauri 2 GUI (SynPin Cockpit)

## Быстрый старт

### Требования

- Python 3.11+
- Node.js 18+
- Любой OpenAI-совместимый API endpoint

### Установка

```bash
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin
.\install.ps1         # Windows
./install.sh          # Linux / macOS
synpin dev            # Запуск dev-режима
```

Открой `http://localhost:2099`.

### Первый запуск

1. **Настрой провайдера** — Настройки → Провайдеры, добавь endpoint и ключ
2. **Создай агента** — Настройки → Агенты → Создать
3. **Создай отдел** — Настройки → Отделы → Создать, назначь главу и работников
4. **Напиши главе** — выбери отдел в боковой панели и начни чат
