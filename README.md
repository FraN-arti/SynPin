## Концепция

SynPin — это платформа, в которой **ИИ-агенты живут как сотрудники организации**, а не как безымянные чат-боты.

У каждого агента есть **имя, роль, место в команде, специализация**. Агенты помнят прошлые разговоры, делят между собой задачи, обсуждают рабочие вопросы — и всё это через единый интерфейс, в одном окне.

Главная идея — **«отдел»**. Это не просто группа агентов, а полноценная структурная единица с:

- **главой отдела** — координатором, который принимает входящие задачи
- **специалистами** — агентами с конкретными ролями (кодер, тестировщик, документалист, архитектор...)
- **правилами взаимодействия** — кто кому делегирует, кто кого слушает
- **общей памятью** — факты, решения, контекст прошлых обсуждений
- **историей чата** — каждое сообщение сохраняется, ничего не теряется

В отличие от обычных ассистентов, здесь агенты **действительно работают вместе** — не по очереди с пользователем, а параллельно друг с другом, по модели близкой к head-delegate из оригинального Hermes Agent, но встроенной в организационную структуру.

## Как работает отдел

Представь, что ты — глава проекта, и тебе нужно собрать фичу. Обычно ты бы пошёл в Jira, написал таск, попросил кого-то, проверил сам, попросил другого, доделал. **В SynPin ты пишешь одному агенту — главе отдела.** Дальше происходит примерно следующее:

```
       ┌─────────────────┐
       │  Ты (пользователь)│
       └────────┬────────┘
                │  "Сделай фичу X"
                ▼
       ┌─────────────────┐
       │   Глава отдела   │  ← Принимает задачу, разбивает
       │   (head agent)  │     на подзадачи, распределяет
       └────┬───┬───┬────┘
            │   │   │
    ┌───────┘   │   └────────┐
    ▼           ▼            ▼
 ┌──────┐  ┌──────┐    ┌──────────┐
 │Кодер │  │Тестер│    │Документалист│  ← Работают параллельно
 └──────┘  └──────┘    └──────────┘
    │           │            │
    └───────────┴────────────┘
                │
                ▼
       ┌─────────────────┐
       │   Глава отдела   │  ← Сводит результаты
       │   собирает ответ │
       └────────┬────────┘
                │
                ▼
       ┌─────────────────┐
       │  Ты получаешь    │  ← Один связный ответ
       │  готовый отчёт   │
       └─────────────────┘
```

**Внутри отдела:**
- Глава **видит** специализацию каждого агента и выбирает, кого привлечь
- Специалисты **помнят** контекст задачи (через общую память отдела)
- Если задача непонятна — глава **уточняет у тебя**, а не делает наугад
- Каждое решение **фиксируется** в истории чата

**Между отделами:**
- Отделы **изолированы** — у каждого свой контекст, своя память, своя история
- Глава одного отдела **не видит** сообщений другого (если ты не дал доступ)
- Задачи между отделами **передаются** через явный API, не через общий чат

Это позволяет держать **независимые рабочие пространства** — например, отдел «Разработка», отдел «Маркетинг», отдел «Поддержка» — каждый со своими агентами и своей динамикой.

## Быстрый старт

### GUI панель управления

Для визуального управления SynPin используйте [**SynPin Cockpit**](https://github.com/FraN-arti/SynPin-Cockpit) — GUI панель с system tray, логами и кнопками запуска.

### Требования

- **Python 3.11+**
- **Node.js 18+** (для веб-интерфейса)
- **git** (для обновлений)
- Любой OpenAI-совместимый API endpoint (9router, Mistral, OpenAI, Anthropic через прокси)

### Установка за 3 шага

```bash
# 1. Клонируем
git clone https://github.com/FraN-arti/SynPin.git
cd SynPin

# 2. Устанавливаем зависимости (проверяет Python, Node, ставит всё нужное)
./install.sh          # Linux / macOS
.\install.ps1         # Windows (PowerShell)

# 3. Запускаем
synpin dev            # Dev-режим: backend + frontend с авто-перезагрузкой
```

Открой `http://localhost:2099` (dev) или `http://localhost:2088` (prod через `synpin start`).

### Первый запуск

1. **Настрой API-ключ** — открой `http://localhost:2099`, перейди в **Настройки → Провайдеры**, добавь свой endpoint и ключ.
2. **Создай агента** — Настройки → Агенты → Создать. Дай имя, опиши роль.
3. **Напиши ему** — выбери агента в боковой панели и начни чат.

### NPM как альтернативный вход (опционально)

Если предпочитаешь npm-обёртку:

```bash
npm install           # Поставит web-зависимости (dev)
npm run dev           # = synpin dev
npm run start         # = synpin start (prod)
npm run doctor        # Проверить, всё ли установлено
npm run update        # Обновить из Git
```
