# 🎭 Agent Roles

## Базовые роли

### 1. 🏛 Архитектор
**Роль:** Проектирование системы
**Цель:** Создать надёжную, масштабируемую архитектуру
**Экспертиза:** Паттерны проектирования, безопасность, масштабируемость
**Инструменты:** Анализ кода, генерация диаграмм, валидация архитектуры

### 2. 💻 Программист
**Роль:** Написание кода
**Цель:** Реализовать архитектуру в код
**Экспертиза:** Языки, фреймворки, best practices, рефакторинг
**Инструменты:** Code execution, file editing, linting

### 3. 🔍 Ревьюер
**Роль:** Проверка качества
**Цель:** Найти ошибки до продакшена
**Экспертиза:** Code review, безопасность, производительность, читаемость
**Инструменты:** Статический анализ, security scan, benchmark

### 4. 🧪 Тестировщик
**Роль:** Тестирование
**Цель:** Убедиться что всё работает
**Экспертиза:** Unit, integration, e2e, load тесты
**Инструменты:** Test runner, assertion framework, mock server

### 5. 🔎 Исследователь
**Роль:** Поиск и анализ информации
**Цель:** Найти лучшие решения, документацию, примеры
**Экспертиза:** Web search, анализ документации, сравнение подходов
**Инструменты:** Web search, API explorer, docs reader

## Структура агента

Каждый агент имеет:

```python
class Agent:
    name: str           # Имя
    role: str           # Роль (architect, programmer, ...)
    goal: str           # Цель
    backstory: str      # Опыт/контекст для промпта
    expertise: list     # Области экспертизы
    tools: list         # Доступные инструменты
    memory: Memory      # Персональная память
    mode: str           # "delegate" | "team" | "group"
```

## Память агента

Каждый агент хранит:
- **Ошибки** — что пошлоwrong и какой урок извлечён
- **Решения** — успешные подходы к проблемам
- **Контекст** — история задач и предпочтения пользователя
- **Знания** — семантический индекс изученного
