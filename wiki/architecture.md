## Слои

### 1. Web UI (`web/`)
React SPA — дашборд для мониторинга и управления:
- Real-time статус агентов и задач
- Просмотр обсуждений в группе
- История ошибок и решений
- Настройка конфигурации (режимы, роли, модели)
- **Технологии:** React 19, Vite 6, TypeScript 5.7, Tailwind 4, Zustand, TanStack Query

### 2. REST API (`core/synpin/api/`)
FastAPI сервер — единая точка входа:
- Отдаёт API endpoints для UI и внешних клиентов
- В продакшене раздаёт собранную React статику (`web/dist/`)
- В dev-режиме (`dev_server.py`) — отдельный Vite dev server с hot-reload
- WebSocket для стриминга событий в реальном времени

### 3. Router (`core/synpin/router/`)
**Мульти-модальный диспетчер** — решает, как выполнить задачу:
- **Делегат** → один агент, без обсуждений
- **Команда** → несколько агентов, обсуждение → консенсус → действие
- **Инженерная группа** → полный цикл с ролями
- Настраивается через конфиг, не хардкод

### 4. Agent Engine (`core/synpin/engine/`)
Основной цикл агентов:
- ReAct-луп (think → act → observe)
- Управление состоянием задач
- Координация между агентами
- Обработка ошибок и retries

### 5. Agents (`core/synpin/agents/`)
Роли агентов — каждый со своей экспертизой:
- Архитектор, Программист, Ревьюер, Тестировщик, Исследователь
- У каждого: промпт, инструменты, память, ограничения

### 6. Tools (`core/synpin/tools/`)
Инструменты агентов:
- MCP-серверы (внешние инструменты)
- Web search, файловая система, выполнение кода
- Легко расширяемые — новый инструмент = новый класс

### 7. Memory (`core/synpin/memory/`)
Система памяти — обязательный компонент:
- **ChromaDB** — векторный семантический поиск по знаниям
- **Per-agent** — каждый агент помнит свои ошибки и решения
- **Shared** — коллективная память команды
- **Error Log** — журнал ошибок для обучения

│   ├── synpin/
│   │   ├── agents/            ← роли агентов
