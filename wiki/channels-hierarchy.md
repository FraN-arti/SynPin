# 🏢 Каналы и иерархия

SynPin подключает **внешние мессенджеры** как каналы для общения с агентами.

---

## Текущая реализация

**Channels = мессенджеры** (Feishu, Telegram, Discord, SMS, Matrix, Yuanbao)

Каждый канал — это подключённая платформа, через которую пользователь общается с агентами.

### Доступные каналы

| Канал | Статус | Описание |
|---|---|---|
| **Local** | ✅ Работает | Прямой доступ к файлам на машине |
| **Feishu (Lark)** | ✅ Работает | Интеграция через WebSocket (channels.yaml) |
| **Telegram** | 🔌 Подключается | Через Telegram Bot API (polling/webhook) |
| **Discord** | 🔌 Подключается | Через Discord Bot (websocket) |
| **WhatsApp** | 🔌 Подключается | Через WhatsApp Business API (webhook) |
| **Slack** | 🔌 Подключается | Через Slack Bot (websocket) |
| **Email** | 🔌 Планируется | Через IMAP/SMTP (polling) |
| **SMS** | 🔌 Планируется | Через SMS провайдер |
| **Matrix** | 🔌 Планируется | Через Matrix protocol |

### Feishu — подключён

```yaml
# ~/.synpin/config/channels.yaml
channels:
  feishu-main:
    name: "Feishu — Основной"
    type: feishu
    status: connected
    app_id: "cli_a5..."
    app_secret: "***"
    mode: websocket         # ← подключение через WebSocket
    binding:
      target: "main"        # привязка к основному агенту
    features:
      notifications: true
      file_upload: true
      commands: true
    rate_limit:
      messages_per_minute: 30
```

## Организационная структура

### Реализовано

```
Уровень 1:  Департаменты (departments.yaml)
             └── Вахтан, Сузумебачи, Волокита, Разработка

             └── Грахатули (общение), Обсуждение аниме
```

## @mention — обязательное обращение

В otdel-чате агенты **обязаны** обращаться друг к другу через `@`:

```
✅ @Иван, можешь проверить мой код?
✅ @Глава, задача готова к ревью
❌ "проверь код" (неясно кому)
❌ "задача готова" (неясно кому сообщают)
```

### Как это работает

1. **Пользователь** пишет `@Иван, ответь`
2. **@mention парсится** — regex `@([A-Za-zА-Яа-яЁё0-9_]+)`
3. **Глава** видит ВСЁ — всегда обрабатывает
4. **Иван** (работник) добавляется в очередь обработки
5. **Глава** может переадресовать: `@Дмитрий, тоже посмотри`
6. **Дмитрий** тоже добавляется в очередь

## Эскалация между отделами (планируется)

Когда Главе/управляющему нужно обратиться в другой отдел — варианты на обсуждение:

1. **Напрямую другому Главе** — личное сообщение между главами
2. **В общий отдел управляющих** — канал где все главы
3. **Таски с пометкой отдела** — задача с указанием целевого отдела

> Решение: вероятно все три варианта, но не сейчас. Определимся при реализации.

## Типы каналов (master для добавления)

channels.yaml содержит `channel_types` — справочник доступных каналов:

| Тип | Обязательные поля | Режимы |
|---|---|---|
| feishu | app_id, app_secret, verification_token | websocket, webhook |
| whatsapp | phone_number_id, access_token, verify_token | webhook |
| telegram | bot_token | polling, webhook |
| slack | bot_token, signing_secret | websocket |
| discord | bot_token | websocket |
| email | imap_host, imap_port, smtp_host, smtp_port, username, password | polling |

### Привязка каналов (binding)

| Цель | Описание |
|---|---|
| `main` | Главный агент системы |
| `department` | Все агенты указанного департамента |
| `agent:<id>` | Конкретный агент по ID |
