# 🏢 Каналы и иерархия

SynPin подключает **внешние мессенджеры** как каналы для общения с агентами.

---

## Текущая реализация

**Channels = мессенджеры** (Feishu, Telegram, Discord, SMS, Matrix, Yuanbao)

Каждый канал — это подключённая платформа, через которую пользователь общается с агентами.

### Доступные каналы

| Канал | Статус | Описание |
|---|---|---|
| **Local** | ✅ Работает | Прямой доступ к файлам на машине |
| **Feishu (Lark)** | ✅ Работает | Интеграция с Feishu/Lark |
| **Telegram** | 🔌 Подключается | Через Telegram Bot API |
| **Discord** | 🔌 Подключается | Через Discord Bot |
| **SMS** | 🔌 Планируется | Через SMS провайдер |
| **Matrix** | 🔌 Планируется | Через Matrix protocol |
| **Yuanbao** | 🔌 Планируется | Через Yuanbao API |

## Планируемая организация (Фаза 3+)

В будущем SynPin поддерживает **организационную структуру** с каналами отделов:

```
Уровень 3:  Совет директоров (Board of Directors)
             └── Стратегия, приоритеты, бюджет
                     │
Уровень 2:  Совет отделов (Heads Council)
             └── Кросс-функциональные задачи, делегирование
                     │
Уровень 1:  Каналы отделов (Department Channels)
             └── Рабочие агенты, выполнение задач
```

### Структура канала (планируется)

```
~/.synpin/data/channels/
├── web-dev/
│   ├── context.md           # что делает этот отдел
│   ├── chat-log.md          # лента сообщений
│   ├── decisions.md         # принятые решения
│   └── members.yaml         # участники + роли
├── api-design/
│   ├── context.md
│   ├── chat-log.md
│   ├── decisions.md
│   └── members.yaml
└── qa-testing/
    ├── context.md
    ├── chat-log.md
    ├── decisions.md
    └── members.yaml
```

### Роли (планируется)

| Роль | Описание |
|---|---|
| **Глава отдела (Head)** | Модерирует, делегирует, создаёт скиллы для команды |
| **Работник (Worker)** | Выполняет задачи, предлагает решения |
| **Директор (Director)** | Стратегия, приоритеты, разрешение конфликтов |

## @mention — обязательное обращение (планируется)

В канале агенты **обязаны** обращаться друг к другу через `@`:

```
✅ @developer, можешь проверить мой код?
✅ @head-web, задача готова к ревью
❌ "проверь код" (неясно кому)
❌ "задача готова" (неясно кому сообщают)
```
