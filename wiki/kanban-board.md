## Философия

SynPin — это **бригада**, а не супер-агент. Где-то есть Fable 5 — один мощный агент который всё делает сам. А SynPin — это команда普通ных агентов, каждый из которых делает своё, а вместе они строят небоскрёб.

**Kanban — это не просто доска задач. Это operating system для AI-компании.**

---

## Концепция

### Глобальная видимость

Доска видна **всем отделам и агентам**. Но задачи назначаются **конкретному отделу**.

```
┌─────────────────────────────────────────────────────────────┐
│                    📋 ГЛОБАЛЬНЫЙ КАНБАН                     │
│                                                             │
│  BACKLOG │ TODO │ IN PROGRESS │ REVIEW │ DONE │ BLOCKED    │
│  ────────┼──────┼─────────────┼────────┼──────┼──────────  │
│          │      │             │        │      │            │
│  [T-003] │[T-001]│  [T-002]   │        │[T-005]│ [T-006]  │
│  MCP     │API   │  Web       │        │Auth  │  Эскалация│
│  Отдел:  │Отдел:│  Отдел:    │        │Отдел:│  Совет    │
│  Реализ. │Web   │  Web+API   │        │API   │  Директ.  │
└─────────────────────────────────────────────────────────────┘
```

### Живой таск

Каждый таск — это **живой документ** с внутренней историей действий:

```markdown
# T-002: Создать MCP интеграцию

**Статус:** In Progress
**Отдел:** Реализации → Web
**Дедлайн:** 2026-06-15
**Приоритет:** High

## История действий

- [2026-06-12 10:00] 📝 Создан Советом Директоров
- [2026-06-12 10:05] 🏢 Принят Главой отдела Реализации
- [2026-06-12 10:10] 📋 Распределён: Агент-1 (интерфейс), Агент-2 (адаптер)
- [2026-06-12 14:00] ✅ Агент-1: Базовый интерфейс готов
- [2026-06-12 16:00] ✅ Агент-2: Адаптер для Claude готов
- [2026-06-12 17:00] 🔍 Глава: Проверка — нет тестов для edge cases
- [2026-06-12 17:05] 🔄 Отправлено на доработку → Агент-2
- [2026-06-12 19:00] ✅ Агент-2: Тесты добавлены
- [2026-06-12 19:30] ✅ Глава: Принято
- [2026-06-12 19:35] 📨 **Summon** → Глава Web-отдела (передано для интеграции на сайт)
- [2026-06-12 20:00] 🏢 Глава Web: Принял, распределяю

## Результаты
- Базовый интерфейс: `/core/synpin/mcp/interface.py`
- Адаптер Claude: `/core/synpin/mcp/adapters/claude.py`
- Тесты: `/core/tests/test_mcp.py`
```

## Жизненный цикл таска

```
Создание          →  Назначение       →  Работа
(Совет Директ.)      (Глава отдела)      (Агенты)
                      ↓                    ↓
                   Проверка  ←────────  Отчёт команды
                      ↓
              ┌───────┴───────┐
              ↓               ↓
         ✅ Принято      ❌ Не соответствует
              ↓               ↓
      Summon / Done      Доработка
                            ↓
                     (Агенты дорабатывают)
                            ↓
                      Проверка снова
                            ↓
                  Если не справляется → Эскалация
```

### Детали этапов

**1. Создание таска**
- Совет Директоров (или человек) создаёт задачу
- Указывает: описание, требуемый отдел, дедлайн, приоритет

**2. Назначение (Глава)**
- Глава отдела получает таск
- Анализирует и распределяет между агентами
- Записывает кто что делает

**3. Работа (Агенты)**
- Агенты выполняют свои части
- Каждое действие записывается в историю таска

**4. Проверка (Глава)**
- Глава сравнивает результаты с требованиями таска
- **Ключевой момент:** это quality gate

**5. Решение: Принять / Доработка / Эскалация**
- ✅ **Принято** → Summon следующему отделу или Done
- 🔄 **Доработка** → Возвращается агентам с конкретными замечаниями
- 🚨 **Эскалация** → Совет Директоров (если отдел не справляется)

## Summon (призыв) — межотделовая передача

Глава одного отдела может **"призвать"** главу другого отдела, передавая таск.

```json
{
  "action": "summon",
  "task_id": "T-002",
  "from_department": "реализации",
  "to_department": "web",
  "reason": "MCP интерфейс готов, нужна интеграция на сайт",
  "context": "Используй адаптер из /core/synpin/mcp/adapters/claude.py"
}
```

**Правила Summon:**
- Таск **перетекает** в новый отдел (не копируется)
- В истории остаётся след: `📨 Summon → Отдел Web`
- Предыдущий отдел **теряет контроль** над таском
- Глава получателя решает — принять или отклонить

## Эскалация в Совет Директоров

Когда отдел **не справляется** с задачей:

```
Глава → "Стоп, не можем" → Формирует отчёт:

📋 Эскалация: T-002
  Отдел: Реализации
  Проблема: Не хватает компетенций для X
  Что сделано: A, B, C
  Чего не хватает: D
  Рекомендация: привлечь отдел Y или изменить требования
```

**Совет Директоров решает:**
- Найти решение самостоятельно
- Перенаправить в другой отдел
- **Эскалировать человеку** (Артуру) — "Написать мне и узнать что делать"

## Колонки доски

| Колонка | Описание | Кто управляет |
|---------|----------|---------------|
| **BACKLOG** | Созданные, но не взятые в работу | Совет Директоров |
| **TODO** | Назначены отделу, ожидание начала | Глава отдела |
| **IN PROGRESS** | Агенты работают | Глава + Агенты |
| **REVIEW** | Работа окончена, проверяется | Глава отдела |
| **DONE** | Задача выполнена и принята | Система |
| **BLOCKED** | Эскалировано, ожидание решения | Совет Директоров |
| **REVISION** | Отправлено на доработку | Глава → Агенты |

## Связь с другими системами

```
                    ┌──────────────┐
                    │  SKILLS/MCP  │ ← Что агенты МОГУТ
                    └──────┬───────┘
                           ↓
┌──────────┐    ┌──────────────────┐    ┌─────────────┐
│  FORUM   │←──→│  ГЛОБАЛЬНЫЙ      │←──→│ ДЕЛЕГИРОВАНИЕ│
│(обсужд.) │    │  КАНБАН          │    │ (главами)   │
└──────────┘    └────────┬─────────┘    └─────────────┘
                         ↓
                ┌────────────────┐
                │ CANVAS СВЯЗЕЙ  │ ← Визуализация
                └────────────────┘
```

- **Kanban + Skills** — таски требуют определённых навыков, глава назначает агента с нужным skill
- **Kanban + Делегирование** — глава распределяет таск через delegate/await
- **Kanban + Forum** — обсуждения происходят в контексте тасков
- **Kanban + Canvas** — визуализация потока тасков между отделами

## Структура данных (проект)

```yaml
# tasks.yaml — глобальный реестр тасков
tasks:
  T-002:
    title: "Создать MCP интеграцию"
    description: "Разработать MCP интерфейс для подключения внешних инструментов"
    department: "реализации"
    assigned_to:
      head: "agent-head-realizaciya"
      workers:
        - agent: "agent-1"
          task: "Базовый интерфейс"
        - agent: "agent-2"
          task: "Адаптер для Claude"
    status: "in_progress"
    priority: "high"
    deadline: "2026-06-15"
    created_at: "2026-06-12T10:00:00"
    history:
      - timestamp: "2026-06-12T10:00:00"
        actor: "council"
        action: "created"
        detail: "Запрос от Совета Директоров"
      - timestamp: "2026-06-12T10:05:00"
        actor: "head-realizaciya"
        action: "accepted"
        detail: "Принял, распределяю"
      - timestamp: "2026-06-12T19:35:00"
        actor: "head-realizaciya"
        action: "summon"
        target_department: "web"
        detail: "Передано для интеграции на сайт"
    results:
      - file: "/core/synpin/mcp/interface.py"
      - file: "/core/synpin/mcp/adapters/claude.py"
      - file: "/core/tests/test_mcp.py"
```

## Команды (будут добавлены)

```bash
synpin tasks list                        # все задачи
synpin tasks list --status in_progress   # по статусу
synpin tasks list --department web       # по отделу
synpin tasks show T-002                  # детали + история
synpin tasks create "Описание" --dept web --deadline 2026-06-15
synpin tasks summon T-002 --to web       # Summon
synpin tasks escalate T-002 --reason "Не хватает компетенций"
```
