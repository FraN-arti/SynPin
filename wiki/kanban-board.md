# 📋 Kanban-доска

Общая доска задач с дедлайнами, историей решений и подписями этапов.

---

## Зачем

Kanban в SynPin — не просто «сделано/не сделано». Это **живая история** каждой задачи:
- Кто создал и зачем
- Какой отдел что сделал
- Какие решения приняты
- Полный путь от Backlog до Done

---

## Подпись этапа — зачем

Когда задача проходит через несколько отделов, **подпись** позволяет:

### 1. Найти источник ошибки

```
Баг: форма не обрабатывает 401

Смотрим подписи:
  @developer (Web) → сделал интеграцию, но без 401
  @reviewer (QA) → заметил, вернул на доработку
  @developer (Web) → исправил

Вывод: ошибка была на этапе разработки, QA сработал ✅
```

### 2. Понять полный путь

```
Задача прошла: heads-council → head-web → developer → reviewer → head-web
5 этапов, 3 отдела, 2 дня
```

### 3. Ответственность

```
Если задача зависла на Review 3 дня:
  → вопросы к @reviewer и главе QA отдела
  → head-web видит что проблема не у него
```

## Команды (будут добавлены)

```bash
synpin tasks list                    # все задачи
synpin tasks list --status todo      # только TODO
synpin tasks list --department web   # задачи отдела Web
synpin tasks show task-2025-06-001   # детали задачи
synpin tasks move task-001 review    # переместить в Review
synpin tasks close task-001          # закрыть задачу
```

---

## Связь с каналами

```
Совет отделов: создаёт задачу → Kanban (Backlog)
     ↓
Глава отдела: берёт задачу → Kanban (Todo) → назначает агентам
     ↓
Агенты: работают → Kanban (In Progress)
     ↓
Ревьюер: проверяет → Kanban (Review)
     ↓
Глава: утверждает → Kanban (Done)
```

Каждое действие в канале автоматически обновляет задачу на доске.

## Авто-очистка

```yaml
# settings.yaml
tasks:
  archive_after: 30d       # архивировать закрытые через 30 дней
  delete_after: 90d        # удалять из архива через 90 дней
  max_active: 50           # максимум активных задач
```
