## Философия

SynPin — это **бригада**, а не супер-агент. Kanban — это не просто доска задач. Это **operating system для AI-компании**.


## Колонки

### Конфигурация

Колонки настраиваются в `columns.yaml`. Каждая колонка имеет:

```yaml
- id: l7r8zs6i26wzyl        # 14-символьный уникальный ID
  label: Backlog             # Отображаемое название
  description: "Ожидает назначения"  # Описание (для промпта агентов)
  color: '#6b7280'           # Цвет индикатора
  order: 0                   # Порядок (определяется drag-reorder)
  enabled: true              # Включена/выключена
```

### Drag-Reorder

На странице "Глобальный Канбан" колонки можно перетаскивать за `⠿` для изменения порядка. Порядок сохраняется автоматически.

### Метки (Labels)

Метки настраиваются в `labels.yaml`:

```yaml
- id: abc123def456gh        # 14-символьный ID
  name: #System              # Отображаемое имя
  color: '#1e3a5f'           # Цвет фона
  text_color: '#93c5fd'      # Цвет текста
```

## Clean Delete

Принцип: **удалил объект → убрал ВСЕ следы**.

### Удаление колонки:
1. Удаляется из `columns.yaml`
2. Удаляется из `widget.yaml` → `show_columns`
3. Таски с этим статусом → переносятся в первую активную колонку

### Удаление метки:
1. Удаляется из `labels.yaml`
2. Тег удаляется из всех задач, которые его использовали

## Жизненный цикл таска

```
Создание          →  Назначение       →  Работа
(Совет Директ.)      (Глава отдела)      (Агенты)
                      ↓                    ↓
                   Проверка  ←────────  Отчёт команды
                      ↓
              ┌───────┴───────┐
              ↓               ↓
         ✅ Принято      ❌ Не соответствует
              ↓               ↓
      Summon / Done      Доработка
                            ↓
                     (Агенты дорабатывают)
                            ↓
                      Проверка снова
                            ↓
                  Если не справляется → Эскалация
```

### Статусы

| Статус | Описание | Цвет |
|--------|----------|------|
| `backlog` | Создано, не взято в работу | ⬜ #6b7280 |
| `todo` | Назначено отделу | 🔵 #3b82f6 |
| `in_progress` | В работе | 🟠 #f97316 |
| `review` | На проверке | 🟡 #f59e0b |
| `revision` | На доработке | 🔴 #ef4444 |
| `blocked` | Заблокировано | 🔴 #dc2626 |
| `done` | Выполнено | 🟢 #22c55e |

### Приоритеты

| Приоритет | Цвет |
|-----------|------|
| `critical` | 🔴 #dc2626 |
| `high` | 🟠 #f97316 |
| `medium` | 🟡 #f59e0b |
| `low` | ⬜ #6b7280 |

## Summon (призыв)

Межотделовая передача таска:

```json
{
  "action": "summon",
  "task_id": "T-002",
  "from_department": "реализации",
  "to_department": "web",
  "reason": "MCP интерфейс готов, нужна интеграция на сайт"
}
```

**Правила:**
- Таск **перетекает** в новый отдел (не копируется)
- В истории остаётся след: `📨 Summon → Отдел Web`
- Предыдущий отдел **теряет контроль**

---

## Эскалация

Когда отдел **не справляется**:

```
Глава → "Стоп, не можем" → Формирует отчёт:
  Отдел: Реализации
  Проблема: Не хватает компетенций для X
  Что сделано: A, B, C
  Рекомендация: привлечь отдел Y
```

**Совет Директоров решает:**
- Найти решение самостоятельно
- Перенаправить в другой отдел
- Эскалировать человеку (Артуру)

## Виджет

Компактный виджет для главной страницы. Настройки в `widget.yaml`:

```yaml
mode: active           # active | all | my | blocked
max_items: 5           # Максимум задач
show_columns:          # Какие колонки показывать
  - in_progress
  - review
  - blocked
show_deadline: true    # Показывать дедлайн
show_department: true  # Показывать отдел
compact: true          # Компактный режим
```

Виджет перетаскивается в зону виджетов из настроек (вкладка "Канбан").

- Компактный список активных задач

## Команды (планируется)

```bash
synpin tasks list                        # все задачи
synpin tasks list --status in_progress   # по статусу
synpin tasks list --department web       # по отделу
synpin tasks show T-002                  # детали + история
synpin tasks create "Описание" --dept web --deadline 2026-06-15
synpin tasks summon T-002 --to web       # Summon
synpin tasks escalate T-002 --reason "Не хватает компетенций"
```
