# 🔧 MCP Integration

## Что такое MCP

**Model Context Protocol** — стандарт Anthropic для подключения инструментов к AI агентам.

- Один протокол для всех инструментов
- Изоляция: каждый MCP-сервер — отдельный процесс
- Агенты вызывают инструменты через единый интерфейс

## Встроенные MCP-серверы

### 1. File System
- Чтение / запись / поиск файлов
- Ограничение по директориям (безопасность)

### 2. Web Search
- Поиск в интернете
- Извлечение контента со страниц

### 3. Terminal
- Выполнение команд
- Управление процессами
- Песочница (ограничения)

### 4. GitHub
- Клониование репозиториев
- Создание PR, issues
- Чтение кода

### 5. Database
- SQL-запросы
- Схема базы данных
- Миграции

## Конфигурация

```yaml
# synpin.yaml
mcp_servers:
  filesystem:
    command: "npx"
    args: ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/project"]
    allowed_dirs: ["/path/to/project"]

  web-search:
    command: "npx"
    args: ["-y", "@anthropic/mcp-server-web-search"]

  terminal:
    command: "python"
    args: ["-m", "synpin.tools.mcp_terminal"]
    sandbox: true
    allowed_commands: ["ls", "cat", "python", "npm"]
```

## Кастомные инструменты

Любой MCP-сервер можно подключить:

```yaml
mcp_servers:
  my-custom-tool:
    command: "python"
    args: ["my_server.py"]
    transport: "stdio"  # или "http"
```

---

## Безопасность

- **Изоляция** — каждый MCP-сервер в отдельном процессе
- **Песочница** — ограничения на команды и директории
- **Whitelist** — разрешённые команды и пути
- **Audit log** — лог всех вызовов инструментов
