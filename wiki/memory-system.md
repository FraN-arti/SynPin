# 🧠 Memory System

## Проблема

В существующих фреймворках:
- Память опциональна
- Каждый запуск с нуля
- Ошибки забываются
- Нет коллективного обучения
- Нет иерархии контекста (координатор vs исполнители)

## Наше решение

**FTS5 + Markdown** — основа. **ChromaDB** — расширение в будущем.

### Почему FTS5 как основа

| Параметр | Значение |
|----------|----------|
| Скорость | Мгновенно (4500× быстрее LLM-поиска) |
| Зависимости | Нулевые — SQLite встроен в Python |
| Формат | Один файл `.db` — portable |
| Стоимость | Бесплатно, локально |
| Для LLM | Структурированный JSON-вывод |

ChromaDB добавим потом как опцию для семантического поиска, когда появится реальная нужда.

## Frozen Snapshot (из Hermes)

Гениальная концепция: **системный промпт фиксируется при старте сессии** и не меняется.

```
Старт сессии
    │
    ▼
Чтение MEMORY.md + USER.md + facts/*.md
    │
    ▼
Заморозка → frozen_snapshot (в system prompt)
    │
    ▼
Во время сессии:
    ├── Записи в MEMORY.md → обновляются на диске
    ├── НО НЕ меняют frozen_snapshot
    └── Tool-ответы показывают актуальное состояние
    │
    ▼
Конец сессии → следующая сессия читает заново
```

**Зачем:**
- Системный промпт стабилен → **prefix cache не ломается**
- Агент видит актуальное состояние через tool-ответы
- Нет race condition при конкурентных записях

## Слои памяти

### 1. MEMORY.md — компактная память

Постоянная память агента. **Не раздувается.**

```markdown
# Memory — architect

## Patterns
- Text-fallback: инструменты шлём как role="user"
- Порты: 2088 prod, 2099 dev
- Гит: всегда советоваться перед push

## Conventions
- YAML для конфигов
- Pydantic v2 для валидации
- TypeScript strict mode в React

## Anti-patterns
- Не забывать auth middleware
- Не делать verbose логи
- Не перегибать с абстракциями
```

**Лимит:** ~2200 символов (как в Hermes). Если не влезает — компакция.

### 2. USER.md — профиль пользователя (глобальный)

Общая для всех агентов. Хранится в `shared/USER.md` (не per-agent).

```markdown
Пользователь — Артур. Полное имя: Артур Калиш.
Работает: fullstack (React/Python), Москва.
Интересы: ИИ/AGI, КИПиА, разработка.
Стиль: без пафоса, делай руками, честность.
```

**Почему глобальный:** все агенты видят одного и того же пользователя. Когда появятся аккаунты — каждый пользователь будет иметь свой USER.md (per-user вместо глобального).

**Лимит:** ~1375 символов.

### 3. facts/*.md — датированные решения

Конкретные ситуативные решения с привязкой ко времени.

```markdown
# 2026-06-03_port-2099

## Контекст
Нужен порт для dev-сервера SynPin

## Решение
Используем 2099 — остальные закрыты

## Статус
Актуально
```

**Почему датированные файлы:** агент при поиске видит актуальность. Факт от вчера > факт от 3 месяцев назад.

**Структура имени:** `YYYY-MM-DD_topic.md`

### 3. sessions/*.md — история диалогов

Полные логи взаимодействий. Источник для компакции и поиска.

### 4. Shared MEMORY.md — коллективное обучение

Ошибка одного агента = знание всех.

## Threat Scanning (из Hermes)

Защита от injection в память.

```python
# При записи в MEMORY.md:
scan_result = threat_scan(content)
if scan_result:
    return error("Blocked: potential injection pattern")
```

Паттерны:
- Prompt injection попытки
- Экзфильтрация данных
- Манипуляция поведением

## Конкурентный доступ (из Hermes)

File locking для безопасной записи из нескольких сессий:

```python
with file_lock(MEMORY.md):
    # Перечитать с диска (другая сессия могла записать)
    fresh = read_file(MEMORY.md)
    # Модифицировать
    fresh.append(new_entry)
    # Записать
    write_file(MEMORY.md, fresh)
```

## Извлечение фактов

Агент не просто ведёт диалог — он **извлекает знания**.

```
Диалог с пользователем
    │
    ▼
Агент маркирует важное:
    📌 "Это важное решение" → facts/YYYY-MM-DD_topic.md
    ⚠️ "Это ошибка" → MEMORY.md (Anti-patterns)
    💡 "Это факт" → MEMORY.md (Patterns)
    │
    ▼
В конце сессии:
    ├── Новые факты → facts/*.md
    ├── Обновления → MEMORY.md (если есть изменения)
    └── Shared → shared/MEMORY.md (если были командные решения)
```

## Поиск

| Что ищем | Где | Как |
|----------|-----|-----|
| «Какой порт для dev?» | facts/*.md | FTS5 по содержимому |
| «Что обсуждали по API?» | teams/*/sessions/*.md | FTS5 по имени файла + содержимому |
| «Какой паттерн для text-fallback?» | MEMORY.md | FTS5, быстрый доступ |
| «Что было 3 месяца назад?» | facts/ (сортировка по дате в имени) | glob + сортировка |

### Операторы поиска

```
python AND fastapi          # оба термина
"exact phrase"              # точная фраза
auth OR login               # любой термин
deploy*                     # wildcard
python NOT java             # исключение
```

## Расширенная память (будущее)

Поверх FTS5 можно подключить векторный поиск:

| Тип | Описание |
|-----|----------|
| **ChromaDB** | Векторный поиск по памяти |
| **Embedding** | Семантический поиск (`all-MiniLM-L6-v2` и др.) |
| **Semantic search** | «Найди похожие ошибки» по смыслу, не по ключевым словам |

Базовый поиск (FTS5) работает всегда. Векторный — опция.
