# 🚀 Onboarding — Стартовое окно

> **📋 Статус: Планируется (Фаза 2)** — это design spec, не реализовано.

> Концепция: первый запуск SynPin встречает пользователя приветствием, а не пустым чатом.

## Проблема

Сейчас при первом запуске:
- Сервер стартует → пользователь видит пустой чат
- Нет провайдеров → агент не отвечает
- Нет инструкций → пользователь не знает что делать

## Решение: стартовое окно

### Условие показа
Окно показывается **один раз** при обнаружении:
- Нет подключённых провайдеров (`providers.yaml` пуст или содержит только `YOUR_API_KEY_HERE`)
- Или флаг `first_run: true` в `settings.yaml`

## Техническая реализация

### Frontend
- Новый компонент `OnboardingWizard.tsx`
- 5 экранов с анимацией переходов
- Состояние: `useState<'wizard' | 'chat'>`
- Проверка first_run при загрузке App

### Backend
- `GET /api/setup/status` — проверка first_run
- `POST /api/setup/complete` — сохранение конфигов, установка `first_run: false`
- `POST /api/setup/test-key` — тест API ключа

### Хранение
```yaml
# ~/.synpin/config/settings.yaml
first_run: false  # true при первом запуске
```

### UI стили
- Dark фон (дефолт SynPin)
- Оранжевые акценты (#f97316)
- 2px radius (стиль SynPin)
- Минимум текста, максимум действия

## Порядок экранов

```
Приветствие → Провайдер → API Key → Модель → Имя → Готово
     ↑                                              │
     └──────── Пропуск (кнопка "У меня есть ключ") ─┘
```

## Связь с компонентами

- `App.tsx` — проверка first_run, рендер OnboardingWizard
- `SettingsPage.tsx` — сохранение провайдеров/моделей
- `MemorySection.tsx` — USER.md заполняется на экране "Имя"
- `store.py` — `_render_block("user", [])` — промпт для первого диалога
