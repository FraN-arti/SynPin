# 🏢 Отделы (Otdels)

SynPin поддерживает **двухуровневую организационную структуру**: Департаменты → Отделы.

---

## Иерархия

```
┌─────────────────────────────────────────────────────────────┐
│                     Департаменты                             │
│  Backend · Frontend · Analytics · DevOps                     │
│                                                             │
│  ┌───────────────────────────────────────────────────────┐  │
│  │                  Отделы (Otdels)                      │  │
│  │                                                       │  │
│  │  Web Team (веб-разработка) ── Глава: agent-001        │  │
│  │  ├── agent-002 (работник)                             │  │
│  │  ├── agent-003 (работник)                             │  │
│  │  ├── agent-004 (работник)                             │  │
│  │  ├── agent-005 (работник)                             │  │
│  │  └── agent-006 (работник)                             │  │
│  │                                                       │  │
│  │  Mobile Team (мобильная разработка) ── Глава: agent-001│  │
│  │  ├── agent-002 (работник)                             │  │
│  │  └── agent-003 (работник)                             │  │
│  └───────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

## Департаменты (departments.yaml)

Департаменты — **высший уровень** специализации агентов.

```yaml
# ~/.synpin/config/departments.yaml
departments:
  - departmentsid: dept-001
    name: Backend
    color: '#f97316'

  - departmentsid: dept-002
    name: Frontend
    color: '#7cf915'

  - departmentsid: dept-003
    name: Analytics
    description: Департамент аналитики и отчётов
    color: '#27166a'

  - departmentsid: dept-004
    name: DevOps
    description: Инфраструктура и деплой
    color: '#bbf73b'
```

### Поля

| Поле | Описание |
|---|---|
| `departmentsid` | Уникальный slug-идентификатор |
| `name` | Отображаемое имя |
| `description` | Описание департамента |
| `color` | Цвет для UI (hex) |
| `mentor_role` | Роль для ментора (опционально) |
| `escalation` | Канал эскалации (опционально) |

## Отделы (otdels.yaml)

Отделы — **второй уровень**, находятся внутри департаментов. Каждый отдел имеет своего **Главу** и **работников**.

```yaml
# ~/.synpin/config/otdels.yaml
otdels:
  - otdelid: otdel-001
    name: Web Team
    description: Команда веб-разработки и фронтенда
    color: '#15f9a2'
    mentor_role: управляющий
    head: agent-001                    # slug агента-Главы
    workers:                          # slug'ы работников
      - agent-002
      - agent-003

  - otdelid: otdel-002
    name: Mobile Team
    description: Мобильная разработка
    color: '#f915db'
    mentor_role: управляющий
    head: agent-001
    workers:
      - agent-002
      - agent-003
      - agent-004
      - agent-005
      - agent-006
    compaction_limit: 100             # лимит сообщений перед компакцией
    keep_recent: 10                   # сколько последних сообщений сохранять
```

### Поля

| Поле | Описание |
|---|---|
| `otdelid` | Уникальный 12-символьный ID |
| `name` | Отображаемое имя отдела |
| `description` | Описание отдела |
| `color` | Цвет для UI (hex) |
| `mentor_role` | Роль для Главы (ссылка на rolesid) |
| `escalation` | Канал эскалации |
| `head` | slug агента-Главы (управляющего) |
| `workers` | Список slug'ов работников отдела |
| `compaction_limit` | Лимит сообщений перед компакцией (по умолчанию 100) |
| `keep_recent` | Сколько последних сообщений сохранять при компакции (по умолчанию 10) |

---

## Роли в отделе

| Роль | Описание | Кто может делегировать |
|---|---|---|
| **Глава (Head)** | Управляет командой, делегирует задачи | Пользователь / Совет директоров |
| **Работник (Worker)** | Выполняет задачи, отчитывается Главе | Глава отдела |
| **Директор (Director)** | Стратегия, приоритеты | Пользователь |

### Текущие роли (roles.yaml)

```yaml
roles:
  - rolesid: управляющий
    name: Управляющий
    description: управляющий отделом
    color: '#f59e0b'

  - rolesid: совет-директоров
    name: Совет Директоров
    description: это почти верхушка айзберга
    color: '#b60af5'

  - rolesid: работник-отдела
    name: Работник отдела
    description: Стандартный агент
    color: '#595245'
```

## Чат отдела

Каждый отдел имеет **свой изолированный чат** с историей сообщений.

### Архитектура

```
┌──────────────────────────────────────────────────────┐
│  WebSocket (/ws)                                     │
│  {"type": "otdel:send", "otdel_id": "...", ...}     │
└──────────────┬───────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────┐
│  WS Router (_handle_otdel_send)                      │
│  ├── Сохраняет сообщение в chat.json                 │
│  ├── Парсит @mentions                                │
│  ├── Глава всегда обрабатывает (видит всё)            │
│  ├── Работники — только при @mention                 │
│  └── Стримит ответы через WS                         │
└──────────────┬───────────────────────────────────────┘
               │
               ▼
┌──────────────────────────────────────────────────────┐
│  Otdel Helpers (otdel_helpers.py)                    │
│  ├── History: chat.json per otdel                    │
│  ├── @mentions: regex @Name                          │
│  ├── System prompt: агент + otdel context + role     │
│  ├── Head context: видит ВСЁ                          │
│  ├── Worker context: видит только своё + Head         │
│  └── Compaction: keep first + last N                 │
└──────────────────────────────────────────────────────┘
```

### API Endpoints

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/api/otdels/{otdel_id}/chat/history` | История сообщений отдела |
| `POST` | `/api/otdels/{otdel_id}/chat/send` | Отправить сообщение (HTTP fallback) |
| `GET` | `/api/otdels/{otdel_id}/chat/task/{task_id}` | Статус задачи (deprecated → WS) |

### WebSocket сообщения

| Тип | Направление | Описание |
|---|---|---|
| `otdel:send` | Client → Server | Отправить сообщение в отдел |
| `otdel:message` | Server → Client | Новое сообщение |
| `otdel:thinking` | Server → Client | Агент начал думать |
| `otdel:chunk` | Server → Client | Стриминг текста |
| `otdel:tool_start` | Server → Client | Начало вызова инструмента |
| `otdel:tool_end` | Server → Client | Завершение инструмента |
| `otdel:done` | Server → Client | Ответ готов |
| `otdel:compacting` | Server → Client | Компакция истории |
| `otdel:error` | Server → Client | Ошибка |

### Поток сообщения

```
Пользователь → "Привет! @Иван ответь"
    │
    ├── Сохраняется в chat.json
    ├── @mention парсится → Иван найден
    │
    ├── Глава: видит всё, обрабатывает
    │   └── Может делегировать через @mention:
    │       "(@Иван, сделай задачу X)"
    │       └── Иван добавляется в очередь
    │
    └── @Иван (работник): обрабатывает задачу
        └── Отвечает: "Задача выполнена"
            └── Глава: follow-up итог для пользователя
```

### Компакция истории

При превышении `compaction_limit` сообщения сжимаются:

```
До:    [msg1, msg2, msg3, ..., msg99, msg100, msg101]
После: [msg1, "[Компакция: удалено 90 сообщений]", msg92, ..., msg101]
```

- Сохраняется **первое** сообщение (контекст)
- Заменяется средняя часть **маркером компакции**
- Сохраняются последние `keep_recent` сообщений

## Роли и департаменты (UI)

### Вкладка "Отделы" в Настройках

1. **Список отделов** — карточки с цветом и описанием
2. **Карточка отдела** — при клике открывается чат
3. **Настройки отдела** — панель с выбором Главы и работников

### Настройки отдела

- **Глава** — выпадающий список агентов с ролью ментора
- **Работники** — чипы с агентами, сгруппированные по департаментам
- **Компакция** — настройки лимита и keep_recent
