# 🎯 Философия SynPin

> *Это не инструмент. Это команда.*

---

## Проблема существующих фреймворков

### delegate_task (Hermes)
- Агент = одноразовый инструмент
- Нет памяти между запусками
- Нет общения с пользователем
- Ошибки забываются

### CrewAI
- Агенты работают последовательно
- Память опциональна
- Dashboard платный ($99/мес)
- Не видно процесс обсуждения

### AG2 (AutoGen)
- Агенты общаются ✅
- Нет встроенной памяти ❌
- Нет дашборда ❌
- Только код, нет UI ❌

## Принципы SynPin

### 1. Память обязательна
Каждый агент запоминает:
- Свои ошибки
- Успешные решения
- Контекст задач
- Предпочтения пользователя

Память — не опция. Это фундамент. Без неё агент каждый раз начинает с нуля.

### 2. Мульти-модальность «как Лего»
Не каждая задача требует обсуждения. Система настраивается:

```
Простой делегат  ←→  Пара агентов  ←→  Инженерная группа
   (задача → результат)    (обсуждение)    (полный цикл)
```

**Router** решает, какой режим нужен:
- **«Напиши тесты»** → делегат (один агент, без обсуждений)
- **«Спроектируй архитектуру»** → команда (архитектор + ревьюер спорят)
- **«Построй микросервис»** → инженерная группа (полный цикл)

Конфигурация настраивается пользователем. Не хардкод.

### 3. Обсуждение перед действием (когда нужно)
Для сложных задач агенты не делают молча:
1. Обсуждают подход
2. Делятся опытом из памяти
3. Приходят к консенсусу
4. Действуют

Для простых — сразу к делу.

### 4. Коллективное обучение
Ошибка одного = знание всех:

```
Агент 1: "Забыл про auth" → Ошибка → Записано в память
Агент 2: "Видел ошибку агента 1" → Не повторяет
Агент 3: "Тоже видел" → Тоже не повторяет
```

### 5. Специализация
Каждый агент — эксперт:
- **Архитектор** — проектирование
- **Программист** — код
- **Ревьюер** — проверка качества
- **Тестировщик** — тесты
- **Исследователь** — поиск и анализ

### 6. Прозрачность
Пользователь видит:
- Кто что делает
- Какие ошибки были
- Как принимались решения
- Какой результат

Dashboard — бесплатный, встроенный, React.

## Техническая философия

### Ядро важнее оболочки
Агентский фреймворк — это интеграции, не raw-производительность.
Python выигрывает по AI-интеграциям в 10 раз, а проигрывает в скорости на 0.005 сек за запрос.
А 99% времени уходит на ожидание LLM-ответа.

### Разделение ответственности
- **Python core** — агенты, память, MCP, оркестрация, REST API
- **React UI** — дашборд, мониторинг, конфигурация
- Можно заменить UI без трогания ядра
- Можно использовать API без UI (мобилка, бот, CLI)

### Локальный-first
- Работает на твоей машине
- Данные не уходят наружу
- LLM через LM Studio / Ollama / любой провайдер
